< Cours 2.1 : feuille de style

Cours / TD 2 : gérer le style en CSS ; les tableaux en html

Objectifs : séparer la gestion du fond et de la forme (style) grâce au CSS ; découvrir les tableaux en HTML et leur appliquer une feuille de style.

1. la gestion des styles par le langage CSS

1.1. Le style est définissable de trois façons

Le style d'un élement d'une page web peut être défini à trois niveaux :

  1. en ligne (inline) ; c'est à dire comme valeurs de style affectées à l'attribut style de la balise de l'élément ciblé ; par exemple :
          <p style="color : lightblue;" >
               Ma prose écrite ici ... <br/>
               bla bla
          </p >
  2. interne à la feuille (inline) ; c'est à dire regroupé dans la balise <title> de la partie <head> :
    <head> :
          <style>
          /* Ceci est un commentaire sur le style */
                p { /* gère le style de tous les paragraphes. */
                   border : solid 2px black;
                   color : lightblue;
                   background-color : orange;
                }
          </style>
    </head> :
  3. externe à la feuille ; c'est à dire dans un ou plusieurs fichiers d'extension .css liés à la page par une balise <link> :
    <head>
          <link rel="stylesheet" type="text/css" href="mystyle.css">
    </head>

1. Les liens simples

1.1. Lien vers une page externe

exemples :

  • lien relatif à une page du même site stockée dans le sous dossier "partie2" du dossier "tuto" :
    <a href="./tuto/partie2/page3.html" target="_self" title="vers le tutoriel">page 3 du tuto partie 2</a>
  • lien absolu vers la page d'accueil (souvent index.html) du site jodenda.free.fr sur le web :
    <a href="http://jodenda.free.fr/" target="_blank" title="vers la page de jodenda">page des TD </a>

Remarques :

  1. l'attribut target est facultatif, pour la valeur _self par défaut, le clic sur le lien remplace la page active par la page de destination ; pour la valeur _blank, un nouvel onglet est ouvert (cela peut allourdir la navigation) ;
  2. l'attribut title est facultatif mais renseigne les robots sur les mots clefs de la page de destination ;

1.2. Lien vers un élément de la même page : vers une ancre

Le code suivant : <a href="#partie2" target="_self" title="vers l'ancre partie2">partie 2 de cette page</a>
, permet de déplacer le focus sur la zone d'affichage d'une balise d'identifiant id = "partie2".

C'est pratique pour créer un sommaire et des retours au début de page (cf. exercices)

1. syntaxe et principes de base

javascript est un langage interprété par le navigateur, son exécution dépend donc de la façon dont est conçue le navigateur, c'est pourquoi il faut utiliser des navigateurs les plus récents possibles.

1.1. Les variables :

Les variables doivent être déclarées à l'aide du mot réservé var

: var maVariable, compteurDeClic;

Le nom des variables est constitué de plusieurs mots accolés selon la convention Camel Case () adaptée : seule la première initiale est minuscule ;

L'affectation (assignation = assign) est réalisée par le signe '=' ; les opérations raccourcies permettent de gagner de la place : maV = mav + 1; peut s'écrire maV += 1;

< Cours 5 : inclure des fichiers audio et vidéo

cours 5 – Les fichiers vidéos et audio

Des fichiers audio et vidéo sont parfois insérés dans les pages internet.
Le W3C recommande d’utiliser le format mp3 pour les fichiers son et le format mp4 pour les fichiers vidéos.

Section 1. Intégration de fichiers VIDEOS

Les fichiers vidéos peuvent être intégrés à l’aide d’une balise spécifique <VIDEO>
Exemple de code :

<video width="400" controls>
     <source src="mov_bbb.mp4" type="video/mp4">
     Votre navigateur de supporte pas la balise VIDEO.
     Pensez à le mettre à jour, bon sang ! bougre d'ectoplasme à roulette !
</video>

Certains attributs permettent de contrôler les caractéristiques de la vidéo. Exemple :

<video width="320" height="240" controls autoplay muted loop >

Dans ce cas, la vidéo proposera une barre de navigation (play, pause, avancement), démarrera dès qu’elle sera chargée (autoplay), le son sera coupé par défaut (muted) et elle recommencera indéfiniment (loop).
Il existe d’autres attributs de cette balise trop mal connue :
Voir : https://www.w3schools.com/tags/tag_video.asp
Les vidéos Youtube peuvent être intégrées dans une page internet avec la balise <IFRAME>. Si elle est peu recommandée pour le référencement, elle offre un intérêt certain pour l’intégration d’une vidéo.
Exemple :

<iframe width="420" height="315" >
     <src="https://www.youtube.com/embed/XGSy3_Czz8k">
</iframe>

Cette balise accepte des attributs qui sont intégrés dans l’URL de la vidéo (P. Desproges) suivante :

<iframe width="560"
        height="315"
        src="https://www.youtube.com/embed/UbDD1NMf_k4?autoplay=1&controls=0&start=12&end=123"
        frameborder="0"
        allow="accelerometer;
        autoplay;
        encrypted-media;
        gyroscope;
        picture-in-picture"
        allowfullscreen>
</iframe>

Pour voir ce que cela donne :

Dans cet exemple, la vidéo ne commence pas seule (autoplay=0), la fenêtre affiche les contrôleurs (controls), démarre à la 12ème seconde et s’arrête à la 15ème.

Enfin, il est possible de prévoir plusieurs formats de vidéo pour s’adapter aux différents navigateurs :

<video controls width="600">
     <source src="iut.mp4">
     <source src="iut.webm">
     <source src="iut.ogg">
     Désolé votre navigateur est trop ancien, faut un peu suivre quoi !
     <p> Voici un lien<a href="video/iut.mp4">pour télécharger cette vidéo sur l’IUT de Rodez</a></p>
</video>

Il est conseillé d’utiliser les formats mp4, webm et ogg (cf. note 1.)

Section II. Intégration de fichiers AUDIOS

Avant le HTML5, il fallait que l’utilisateur télécharge un plug-in pour écouter des fichiers sons. Les navigateurs récents permettent d’utiliser la balise AUDIO qui évite ces problèmes(cf. notes 2.).
Un fichier audio peut être intégré dans une page internet avec le code suivant :

<audio controls>
     <source src="horse.mp3" type="audio/mpeg">
     Votre navigateur était utilisé par les australopithèques, changez-le !
</audio>

Comme la balise VIDEO, la balise AUDIO accepte certains attributs :

<audio controls autoplay loop>
     <source src="horse.mp3" type="audio/mpeg">
</audio>

Dans ce cas, le fichier affichera des contrôles pour naviguer dans le fichier (controls), démarrera seul (autoplay) et tournera en boucle (loop).
Ainsi, pour avoir un fond sonore à votre page internet, il suffit de ne pas mettre l’attribut controls :

<audio autoplay loop>
     <source src="horse.mp3" type="audio/mpeg">
</audio>

Mais cette pratique est mal supportée par les internautes…

Comme pour les vidéos, il est possible de prévoir plusieurs formats audios :

<audio controls>
     <source src="horse.ogg" type="audio/ogg">
     <source src="horse.mp3" type="audio/mpeg">
Pensez à mettre à jour votre navigateur…
</audio>

Plus d’attributs de la balise AUDIO : balise <audio> sur w3school

  2. Pour être compatible avec les anciens navigateurs, il faut alors utiliser des scripts JS pour repérer le type de navigateur et proposer une solution alternative (téléchargement, mise à jour, etc.).
